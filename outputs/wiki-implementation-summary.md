# Внедрение wiki для пользовательской документации

Подготовлен стартовый каркас решения для пользовательской wiki на субдомене с автоматическим обновлением по изменениям в основном репозитории продукта.

## Что уже сделано

- создан каркас сайта на `MkDocs`
- добавлена базовая структура пользовательских разделов
- введена карта функций `docs-map/features.json`
- добавлена карта сценариев для автоскриншотов
- реализован скрипт определения затронутых функций по изменённым файлам
- реализован скрипт генерации страниц функций
- реализован скрипт построения плана скриншотов
- добавлен workflow wiki-репозитория для автоматического обновления через PR
- добавлен пример workflow для продуктового репозитория

## Что нужно подключить следующим шагом

1. Заполнить реальные `codePaths` и роли в `docs-map/features.json`
2. Подключить staging-стенд и учётную запись для автоскриншотов
3. Реализовать рабочий Playwright-сценарий вместо примера
4. Настроить GitHub token для `repository_dispatch`
5. Настроить деплой wiki на целевой субдомен

## Принцип сопровождения

Изменения в коде не публикуют документацию напрямую. Они создают обновление в виде pull request, чтобы команда могла проверить текст и изображения перед публикацией.
