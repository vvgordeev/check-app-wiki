# Product Wiki Starter

Этот репозиторий содержит стартовый каркас пользовательской wiki по продукту с автоматическим обновлением документации по изменениям в основном репозитории приложения.

## Что внутри

- `mkdocs.yml` - конфигурация wiki-сайта
- `docs/` - пользовательская документация
- `docs-map/` - карта соответствия функций, кода, статей и скриншотов
- `scripts/` - генерация страниц и определение затронутых функций
- `.github/workflows/` - CI для синхронизации wiki с продуктовым репозиторием
- `examples/` - пример workflow для основного репозитория продукта

## Целевая схема

1. Основной репозиторий продукта отправляет событие или запускает workflow при merge.
2. Скрипт определяет, какие функции затронуты изменёнными файлами.
3. Для этих функций пересобираются страницы и при необходимости запускаются автоскриншоты.
4. В репозитории wiki создаётся pull request с обновлёнными статьями и изображениями.
5. После проверки PR wiki публикуется на субдомене.

## Быстрый старт

1. Заполнить `docs-map/features.json`.
2. Актуализировать страницы в `docs/features/`.
3. Подключить workflow из `.github/workflows/docs-sync.yml`.
4. Настроить публикацию сайта на `wiki.<ваш-домен>`.
5. Добавить в продуктовый репозиторий пример из `examples/product-repo.notify-wiki.yml`.

## Принцип ведения

Текст не должен генерироваться полностью "из воздуха". Источником правды остаются:

- карта функций
- шаблоны страниц
- сценарии экранов
- скриншоты со staging-стенда

Автоматизация готовит черновик, а публикация идёт через review.
